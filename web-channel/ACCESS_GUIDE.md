# Web Channel 访问指南

## 🌐 网络环境说明

当前运行在 QEMU 虚拟机中，网络配置为 NAT 模式：
- **虚拟机 IP**: 192.168.122.217
- **宿主机网络**: 与虚拟机不在同一网段
- **访问限制**: 无法直接从宿主机访问虚拟机 IP

## ✅ 可用的访问方式

### 方式 1: 在虚拟机内部访问（推荐）

直接在虚拟机浏览器中访问：
```
http://localhost:5000
或
http://127.0.0.1:5000
```

**步骤**:
1. 在虚拟机中打开浏览器
2. 访问 `http://localhost:5000`
3. 开始使用

### 方式 2: 使用 SSH 端口转发（从宿主机）

在**宿主机**上运行以下命令：
```bash
ssh -L 5000:localhost:5000 yl@192.168.122.217
```

然后在宿主机浏览器访问：
```
http://localhost:5000
```

**注意**: 需要 SSH 服务在虚拟机上运行，且 SSH 密码/密钥已配置。

### 方式 3: 使用 QEMU 端口转发（需要宿主机配置）

在宿主机上配置端口转发规则（需要 QEMU/KVM 权限）：
```bash
# 将宿主机的 5000 端口转发到虚拟机的 5000 端口
sudo iptables -t nat -A PREROUTING -p tcp --dport 5000 -j REDIRECT --to-port 5000
```

### 方式 4: 使用桥接网络（需要重新配置虚拟机）

将虚拟机网络改为桥接模式，使虚拟机获得与宿主机同一网段的 IP。
